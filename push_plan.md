# 推送前清单

## 目标

把本轮改动整理成可推送状态，并确保升级包、说明文档、测试结果和版本口径一致。

## 要产出的内容

### 升级包

- 生成升级包
- 生成升级包校验文件
- 确认升级包目录结构正确
- 确认升级包里只包含需要的文件
- 确认升级包不包含运行数据、数据库、历史指标或敏感信息

### 升级包说明

- 写清楚升级包组成
- 写清楚必填文件
- 写清楚可选文件
- 写清楚禁止包含的内容
- 写清楚升级执行者
- 写清楚升级流程
- 写清楚前置条件

### 升级范围说明

- 写清楚支持哪些升级路径
- 写清楚哪些路径允许同版本重同步
- 写清楚哪些路径需要累计迁移
- 写清楚哪些旧版本不支持原地升级
- 写清楚兼容范围和不兼容范围

### 文档更新

- 更新 README
- 更新中文 README
- 更新升级中心设计文档
- 更新版本治理文档
- 更新部署文档
- 更新升级问题说明
- 更新 release 说明
- 更新计划、发现、进度类记录

## 版本口径检查

- 检查正式版本口径是否已经统一
- 检查文档里是否还有旧的当前版本表述
- 检查测试包示例是否和正式口径混淆
- 检查升级包说明里是否把测试目标版本写成正式版本
- 检查版本变更是否只发生在该变更的文档和代码里

## 验证项

- 后端测试
- 前端测试
- 升级包构建检查
- 文档链接和语法检查
- `git diff --check`

## 推送前判断

- 看这次改动要推到哪个仓库
- 看这个仓库当前应该用哪个分支
- 看是否需要同步到其他分支
- 看是否需要额外打标签
- 看是否需要同时更新 release 说明

## 推送动作

- 先把本地改动整理成可提交状态
- 再按当前仓库和分支要求提交
- 如果需要同步其他分支，再按对应仓库规则处理
- 如果需要发版，再补 tag 和 release 说明

## 注意

- 这里不固定写某一个版本号。
- 这里不默认主分支。
- 这里不默认某一个仓库。
- 文档里要注意版本变更，避免把测试包目标版本写成正式版本。
