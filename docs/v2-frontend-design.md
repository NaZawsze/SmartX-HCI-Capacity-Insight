# v2 前端设计

更新时间：2026-06-06

## 1. 设计原则

v2 保留 v1 信息架构，降低用户学习成本。页面内部可以重构，但导航、主要入口和核心工作流不做大幅迁移。

原则：

- 前端视觉风格必须和 v1 保持一致，延续现有蓝白业务风格、卡片密度、按钮形态、导航结构和客户交付感。
- 可以重构组件代码，但不能把产品改成全新的视觉语言。
- 首页先回答“有没有容量风险”。
- 报表页服务客户交付。
- 服务管理集中处理与集群无关的运维能力。
- 所有后台任务都进入右上角任务中心。
- 上传、预检查、备份、升级、清理都使用页面内进度，不使用浏览器原生弹窗。

不允许：

- 不做营销落地页风格。
- 不做大面积渐变、装饰图形或与当前产品不一致的视觉主题。
- 不改变 v1 用户已经熟悉的主导航和主要操作位置。
- 不为了重写而改变业务术语。

## 2. 主导航

导航顺序：

```text
Dashboard
虚拟机
报表
设置
服务管理
```

全局区域：

- 左侧导航。
- 顶部 scope selector。
- 右上角任务中心。
- 右上角 admin 头像菜单。

服务管理与集群 scope 无关，进入服务管理时页面内容居中，不依赖集群侧栏。

## 3. 通用组件

| 组件 | 用途 |
| --- | --- |
| `ScopeSelector` | 全部/Tower/集群切换 |
| `TaskCenter` | 后台任务列表、进度、日志、下载 |
| `UploadPanel` | 升级包、迁移包上传 |
| `StepProgress` | 预检查、备份、升级步骤 |
| `DataTable` | 支持排序、筛选、空状态 |
| `RiskCard` | 容量风险和告警摘要 |
| `TrendChart` | VM 和集群容量趋势 |
| `ArtifactLink` | 报表、迁移包下载 |
| `ConfirmModal` | 危险操作确认和进度展示 |

交互规则：

- 下拉菜单点击空白处自动收起。
- 按钮 loading 时禁用重复点击。
- 上传显示速度；上传结束但后端仍处理时显示“服务器处理中”。
- 步骤状态：未开始为空心圆，运行中为 loading，成功为绿色勾，失败为红色 X。
- 清理类操作先扫描，再确认，再清理。

## 4. Dashboard

页面目标：一眼看出当前是否有容量风险。

模块：

- 容量风险卡片。
- Tower 数量。
- 集群数量。
- VM 数量。
- 容量使用率。
- SmartX ZBS 容量卡。
- 采集状态。
- 日增长最快 VM。
- 本日新建 VM。
- 集群容量列表。

规则：

- 任一集群 `>=80%` 显示高风险。
- 任一集群 `75%-80%` 显示需关注。
- 无风险显示 `当前所有集群暂无明显容量风险`。
- 本日新建 VM 是独立卡片，放在日增长最快 VM 下方。
- VM 条目点击跳转 VM 页面并定位。

## 5. VM 页面

页面目标：按 scope 查看 VM 容量和趋势。

模块：

- VM 列表。
- 搜索和排序。
- 趋势窗口：7/14/30/90/180/365 天。
- VM 详情。
- 卷详情。

规则：

- VM 展示名称使用最新采集名称。
- 趋势查询使用 `tower_id + cluster_id + vm_id`。
- 删除重建同名 VM 视为新 VM。
- 改名 VM 历史趋势不断裂。

## 6. 报表页

页面目标：页面预测和客户导出共用同一数据口径。

模块：

- 集群预测报表。
- 统计窗口：7/30/90/365/720 天。
- 90 天预测。
- 7 天平均容量增长速率。
- 日增长最快 VM。
- 本日新建 VM。
- 月增长最快 VM。
- 本月新建 VM。
- 导出按钮。

规则：

- 月增长榜只展示样本跨度满 30 天的 VM。
- 刚部署不足 30 天且无满足样本时，月增长榜为空。
- 本日/本月新建 VM 都是独立卡片，不嵌入增长榜。
- 导出任务进入任务中心，完成后提供 Word/Excel 下载。
- 导出文件名包含 scope、日期时间和统计天数。

## 7. 设置页

页面目标：只处理 Tower 和集群配置。

模块：

- Tower 列表。
- Tower 新增/编辑。
- 连接测试。
- 集群同步。
- 集群启用/禁用。

不放：

- 数据迁移。
- 服务重启。
- 系统升级。
- 空间清理。

这些都属于服务管理。

## 8. 服务管理页

页面目标：集中处理与集群无关的运维能力。

二级菜单：

```text
数据迁移
系统运维
升级中心
空间清理
```

数据迁移：

- 迁出。
- 迁入。
- 导入前备份提示。
- 任务进度。
- 下载链接。

系统运维：

- 重启数据服务。
- 显示重启结果和服务状态。

升级中心：

- 上传升级包。
- 包列表。
- 预检查。
- 开始升级。
- 回滚。
- 历史记录。

空间清理：

- 扫描升级包、报表、迁移包、导入留档、旧镜像。
- 显示列表、大小、预计释放。
- 用户确认后清理。
- 显示清理进度和日志。

## 9. 响应式规则

- 主要内容最大宽度适中，服务管理页面居中。
- 卡片在移动端纵向堆叠。
- 表格允许横向滚动。
- 按钮组自动换行。
- 文本不能溢出按钮或卡片。
- 不使用单一色系铺满页面；保持现有蓝白业务风格。

## 10. v1 风格继承清单

v2 实施前端时必须对照 v1 页面保持一致：

- 左侧主导航的位置、信息层级和选中态。
- 顶部工具区、任务中心和 admin 头像菜单位置。
- Dashboard 指标卡、SmartX ZBS 容量卡、风险提示的整体信息密度。
- 报表页增长榜、预测卡、导出按钮的业务化呈现方式。
- 服务管理页的 CloudTower 风格大面板、左侧二级菜单和右侧内容区。
- 上传、预检查、升级步骤、空间清理弹窗的页面内反馈风格。
- 图表、表格、状态图标、按钮颜色必须延续 v1 已形成的蓝白主色和绿色/黄色/红色状态语义。
